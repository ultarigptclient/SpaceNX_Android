Flow: 채팅 송수신

내가 메시지를 보낼 때와 상대방이 보낸 메시지를 받을 때의 양방향 플로우를 봅니다. 핵심은 "내가 보낸 경우도 서버 Push를 기다리지 않고 optimistic하게 DB/UI 반영" 전략입니다.

A. 송신 플로우

sequenceDiagram autonumber participant U as 사용자 participant JS as React ChatPage participant B as Bridge participant H as ChatHandler participant REST as REST participant DB as chat.db participant S as Server participant P as 상대방 U->>JS: 메시지 입력 + 전송 JS->>JS: optimistic UI update
(pending 상태로 말풍선 표시) JS->>B: postMessage({action:"sendChat", channelCode, contents, ...}) B->>H: ChatHandler.handleSendChat() H->>REST: POST /api/comm/sendchat REST->>S: 서버 처리 S-->>REST: {errorCode:0, chatCode, sendDate} REST-->>H: 응답 H->>DB: chatDao.upsert(new ChatEntity) H->>DB: channelDao.updateLastChat(channelCode, contents, sendDate) H-->>JS: resolveToJs("sendChat", {chatCode, sendDate}) JS->>JS: pending → confirmed로 전환 Note over S,P: 서버가 채널 참여자 모두에게 Push S-->>P: Binary SEND_CHAT_EVENT (0x0280) P->>P: PushEventHandler → DB 반영
→ window.postMessage({event:"neoPush",
command:"SendChatEvent", data}) P->>P: React: 새 말풍선 렌더링

B. 수신 플로우 (Push)

sequenceDiagram participant S as Server participant SM as SocketSessionManager participant PH as PushEventHandler participant DB as chat.db participant JS as React S-->>SM: Binary SEND_CHAT_EVENT
(invokeId=0) SM->>SM: isPushEvent(0x0280) → PushEventHandler로 SM->>PH: handleSendChatEvent(data) PH->>DB: chatDao.upsert(ChatEntity.fromJson(data)) PH->>DB: channelDao.updateLastChat + updateUnreadCount PH->>JS: forwardPushToReact("SendChatEvent", data)
→ window.postMessage({event:"neoPush", ...}) JS->>JS: 이벤트 리스너에서 수신
활성 채널이면 즉시 렌더링
아니면 채널 배지만 갱신

읽음 처리 플로우

sequenceDiagram participant JS participant B participant H as ChatHandler participant REST participant DB as chat.db participant S as Server participant Peer as 발신자 JS->>B: postMessage({action:"readChat", channelCode, chatCode}) B->>H: handleReadChat() H->>REST: POST /api/comm/readchat
{channelCode, chatCode, userId} REST-->>H: {errorCode:0, offsetDate} H->>DB: channelOffsetDao.upsert(offset) H-->>JS: resolveToJs("readChat") Note over S,Peer: 서버가 발신자에게 읽음 알림 S-->>Peer: Binary READ_CHAT_EVENT (0x0281) Peer->>Peer: PushEventHandler → offsetDate 갱신
unreadCount 재계산 Peer->>Peer: React: 말풍선 옆 읽음 뱃지 업데이트

private val pendingReads = Collections.synchronizedSet(mutableSetOf<String>())

private suspend fun handleReadChat(params: Map<String, Any?>) {
    val key = "${channelCode}:${chatCode}"
    if (!pendingReads.add(key)) return      // 이미 진행 중

에러/재시도

  • 네트워크 실패: rejectToJs("sendChat", e.message). React가 pending 말풍선을 "전송 실패" 표시
  • 사용자가 재시도: 동일 sendChat 액션 재호출
  • 서버 에러(errorCode ≠ 0): 메시지에 따라 토스트 표시
