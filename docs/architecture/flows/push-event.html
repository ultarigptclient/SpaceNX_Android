Flow: Push 이벤트 처리

서버에서 오는 실시간 이벤트는 두 경로를 탑니다: 앱이 실행 중일 때는 Binary Socket, 백그라운드/종료 상태일 때는 FCM. 둘 다 최종적으로 로컬 DB에 반영되고 (앱이 떠있으면) WebView로 전달됩니다.

2. Binary Socket Push 경로

invokeId == 0인 프레임은 Push 이벤트로 분류되어 PushEventHandler로 라우팅됩니다.

sequenceDiagram participant S as Server participant C as BinarySocketClient participant SM as SocketSessionManager participant PEH as PushEventHandler participant DB participant Web as WebView participant GN as GroupNotificationManager S-->>C: [cmd:2B][len:3B][invokeId=0][body] C->>SM: onMessage(cmd, body) SM->>SM: ProtocolCommand.isPushEvent(cmd) == true SM->>PEH: handlePushEvent(cmd, body) alt 채팅 이벤트 (0x0280~0x028F) PEH->>PEH: handleSendChatEvent(data) PEH->>DB: chatDao.upsert + channelDao.updateLast PEH->>Web: forwardPushToReact("SendChatEvent", data) opt activeChannelCode ≠ 받은 채널 PEH->>GN: 시스템 알림 표시
(인앱 알림 억제) end else Org 이벤트 (0x0380~0x0383) PEH->>DB: userDao / deptDao upsert PEH->>Web: forwardPushToReact("OrgUserEvent", data) else Presence (0x0680 ICON_EVENT) PEH->>Web: _onPresenceUpdate(data) else 기타 (알림, 쪽지, 프로젝트, ...) PEH->>DB: 해당 DB 업데이트 PEH->>Web: forwardPushToReact(command, data) end

4. PushEventHandler 분기 구조

class PushEventHandler(
    private val dbProvider: DatabaseProvider,
    private val channelRepo: ChannelRepository,
    private val messageRepo: MessageRepository,
    private val notiRepo: NotiRepository,
    private val appConfig: AppConfig
) {
    suspend fun handlePushEvent(command: Int, data: JSONObject) {
        when (command) {
            ProtocolCommand.SEND_CHAT_EVENT.code      -> handleSendChatEvent(data)
            ProtocolCommand.READ_CHAT_EVENT.code      -> handleReadChatEvent(data)
            ProtocolCommand.DELETE_CHAT_EVENT.code    -> handleDeleteChatEvent(data)
            ProtocolCommand.MOD_CHAT_EVENT.code       -> handleModChatEvent(data)
            ProtocolCommand.REACTION_CHAT_EVENT.code  -> handleReactionChatEvent(data)

            ProtocolCommand.SEND_MESSAGE_EVENT.code   -> handleSendMessageEvent(data)
            ProtocolCommand.RETRIEVE_MESSAGE_EVENT.code -> handleRetrieveMessageEvent(data)

            ProtocolCommand.NOTIFY_EVENT.code,
            ProtocolCommand.NOTIFICATION_EVENT.code   -> handleNotifyEvent(data)

            ProtocolCommand.ICON_EVENT.code           -> handleIconEvent(data)

            ProtocolCommand.ORG_USER_EVENT.code,
            ProtocolCommand.ORG_DEPT_EVENT.code       -> handleOrgEvent(command, data)

            // ... 30+ 이벤트 타입
        }
    }
}

6. 인앱 알림 억제

사용자가 현재 보고 있는 채널에는 시스템 알림을 띄우지 않습니다:

private suspend fun handleSendChatEvent(data: JSONObject) {
    val channelCode = data.optString("channelCode")
    // DB 반영
    dbProvider.getChatDatabase().chatDao().upsert(ChatEntity.fromJson(data))

    // React로 전달 (항상)
    forwardPushToReact("SendChatEvent", data)

    // 시스템 알림은 활성 채널이 아닐 때만
    if (ctx.activeChannelCode != channelCode) {
        val isMuted = appConfig.isChannelMuted(channelCode)
        if (!isMuted) {
            groupNotificationManager.show(channelCode, senderName, contents)
        }
    }
}

activeChannelCode는 Bridge focusChannel/setActiveChannel 액션으로 React가 갱신합니다.

7. additional null 방지 (React crash 방지)

fun forwardPushToReact(command: String, data: JSONObject) {
    // React에서 null.fileName 접근으로 크래시 방지
    if (!data.has("additional")) data.put("additional", JSONObject())
    val pushMsg = JSONObject()
        .put("event", "neoPush")
        .put("command", command)
        .put("data", data)
    evalJsMain("window.postMessage('${esc(pushMsg.toString())}')")
}
