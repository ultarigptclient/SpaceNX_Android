5. DI가 되지 않는 것들 (개선 대상)

현재 Hilt 주입 안 되는 클래스들
  • Bridge Handler 8종 — BridgeDispatcher에서 new AppHandler(...)로 직접 생성. Context 인터페이스로 의존성을 전달받지만, Hilt @Inject로 전환하면 테스트/모킹이 쉬워짐.
  • ApiClient — Kotlin object (싱글톤). Retrofit 인스턴스를 @Provides로 내보내면 더 깔끔.
  • WebAppBridge — MainActivity에서 직접 new.
개선 시 @InstallIn(ActivityComponent::class) 또는 ViewModelComponent 스코프 검토.

6. 리팩토링 제안

6-1. AppConfig 분리

현재 354줄로 과적재 상태. 3개로 분할 권장:

  • CredentialManager — EncryptedSharedPreferences (토큰/비번)
  • EndpointResolver — REST/소켓 URL 결정
  • ConfigCache — 서버 설정(syncConfig 응답) 캐시

6-2. Retrofit 모듈 분리

별도 @Module NetworkModule에 OkHttpClient, Retrofit, 인터페이스 Provider 이동:

@Module
@InstallIn(SingletonComponent::class)
object NetworkModule {
    @Provides @Singleton
    fun provideOkHttp(): OkHttpClient = OkHttpClient.Builder()
        .addInterceptor(JwtAuthInterceptor(/* appConfig */))
        .build()

    @Provides @Singleton
    fun provideRetrofit(okHttp: OkHttpClient, config: AppConfig): Retrofit = Retrofit.Builder()
        .baseUrl(config.getRestBaseUrl())
        .client(okHttp)
        .addConverterFactory(GsonConverterFactory.create())
        .build()

    @Provides @Singleton
    fun provideAuthApi(retrofit: Retrofit): AuthApi = retrofit.create(AuthApi::class.java)
    // ... 나머지 6개도 동일 패턴
}
