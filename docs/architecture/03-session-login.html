03. 세션 & 로그인 흐름

로그인 종류 2가지(수동/자동), JWT 관리, HI 핸드셰이크, 재연결 정책까지 세션과 관련된 모든 내용을 한 페이지에 정리합니다.

1. LoginState — 단일 진실 소스

로그인 상태는 LoginViewModel의 StateFlow<LoginState>가 유일한 진실 소스입니다. Activity는 이걸 구독만 합니다.

sealed class LoginState {
    object Idle : LoginState()
    object InProgress : LoginState()
    data class Authenticated(val userJson: JSONObject) : LoginState()   // 즉시 UI 진입
    data class LoggedIn(val sessionData: JSONObject) : LoginState()     // HI 완료 후
    data class Failed(val errorCode: Int, val errorMessage: String) : LoginState()
    object LoggedOut : LoginState()
}
Authenticated vs LoggedIn의 차이
Authenticated는 REST 로그인 성공 직후 방출되어 UI에 즉시 SPA를 로드하게 합니다. LoggedIn은 Binary HI 핸드셰이크 완료 후 방출되어 "이제 실시간 Push도 받을 수 있다"는 신호로 쓰입니다.

3. 자동 로그인 (reconnect)

앱 재시작 시 startProcess()에서 저장된 refreshToken + userId 확인 후 자동 로그인을 시도합니다.

sequenceDiagram participant App as MainActivity.startProcess() participant VM as LoginViewModel participant A as AuthRepository participant REST as REST participant S as Socket participant JS as WebView App->>App: 저장된 refreshToken + userId 확인 alt 있음 App->>VM: reconnect(userId) VM->>A: reconnect(userId) A->>REST: POST /api/auth/refresh (최대 3회, 5초 간격) REST-->>A: 새 accessToken A->>A: 새 토큰 저장 A->>A: userJson 구성 (저장된 userId/userName/deptId) A-->>VM: ★ LoginState.Authenticated 즉시 방출 App->>JS: webView.loadUrl(spaUrl) par 백그라운드 A->>REST: POST /api/auth/syncconfig Note right of A: SKIN 변경 감지 시 SkinChanged 이벤트 and A->>S: connectSocket() end else 없음 App->>JS: webView.loadUrl(spaUrl) — 로그인 화면 표시 end

저장된 로그인 정보 복원 (_restoreSavedLogin)

로그아웃 후 로그인 화면에 진입하면 Android가 자동으로 ID를 채워줍니다:

// 로그인 화면 진입 시
val saved = JSONObject().apply {
    put("userId", appConfig.getSavedUserId())
    put("password", if (appConfig.shouldSavePassword()) appConfig.getSavedPassword() else "")
    put("savePassword", appConfig.shouldSavePassword())
    put("autoLogin", appConfig.shouldAutoLogin())
}
evalJs("window._restoreSavedLogin && window._restoreSavedLogin('${esc(saved.toString())}')")

4. JWT 관리

4-1. 저장 위치

모든 토큰/비밀번호는 EncryptedSharedPreferences (AES256-GCM)에 저장. 관리는 AppConfig가 담당.

키용도
KEY_ACCESS_TOKENJWT access token — REST 요청 헤더
KEY_REFRESH_TOKENJWT refresh token — 만료 시 갱신용
KEY_USER_ID로그인 ID (자동로그인용)
KEY_PASSWORD사용자가 "비밀번호 저장" 체크 시 보존
KEY_SAVE_PASSWORD, KEY_AUTO_LOGIN토글 상태

5. HI 핸드셰이크 (Binary Socket 초기화)

Socket 연결 후 가장 먼저 보내는 프레임. 이게 성공해야 소켓으로 다른 명령을 보낼 수 있습니다.

// SocketSessionManager.sendHI()
val body = JSONObject().apply {
    put("userId", userId)
    put("token", jwtToken)
    put("deviceType", "android")
    put("pushToken", fcmToken)
    put("deviceId", deviceId)
    put("appVersion", BuildConfig.VERSION_NAME)
}
binarySocketClient.sendFrame(ProtocolCommand.HI.code, body.toString().toByteArray())

응답 처리

  • errorCode == 0 확인
  • 서버가 새 토큰을 발급하면 저장 (서버는 때때로 토큰 갱신을 HI 응답으로 실어 보냄)
  • jwtTokenDeferred.complete(), hiCompletedDeferred.complete()
  • LoginState.LoggedIn 방출
  • TIME_REQUEST (0x0A07) 송신 (시각 동기화)
CompletableDeferred 패턴
jwtTokenDeferred, hiCompletedDeferred는 "토큰이 준비될 때까지" / "HI가 완료될 때까지" 대기하는 다른 코루틴의 동기화 수단입니다. Socket 프레임을 보내기 전 await()하는 패턴이 여러 곳에 있습니다.

6. 재연결 정책

BinarySocketClient는 연결이 끊기면 backoff 재연결을 시도합니다.

트리거동작
WebSocket close (정상)즉시 재연결 시도
IOException (비정상)지수 백오프 (최대 30초)
Keep-alive NOOP 실패연결 종료 → 재연결
HI 실패토큰 갱신 후 재시도, 3회 실패 시 로그아웃

7. 로그아웃 흐름

sequenceDiagram participant JS as React participant B as Bridge participant D as Dispatcher participant A as AuthRepository participant S as Socket JS->>B: postMessage({action:"logout"}) B->>D: dispatch("logout") D->>A: requestLogout() A->>S: 소켓 TCP 종료 A->>A: 캐시 초기화 (Org/Channel/Buddy/UserNameCache) A->>A: HTTP 클라이언트 해제 A->>A: accessToken / refreshToken 삭제 A-->>JS: window.postMessage({event:"neoDisconnected"}) A-->>JS: window._logoutResolve() A-->>JS: window._restoreSavedLogin({userId, password, ...}) Note over JS: 로그인 화면으로 전환

저장된 userId/password는 삭제되지 않습니다(토글 상태에 따라). 로그아웃 후 로그인 화면에 _restoreSavedLogin으로 다시 채워줍니다.

8. 세션 수명 스코프

AuthRepository / SocketSessionManager는 다음 세 가지 스코프를 다룹니다:

스코프생존 기간용도
applicationScope앱 프로세스 전체DB 워밍업, FCM 토큰
loginSessionScopeAuthenticated → LoggedOutSocket, sync, Push 이벤트 처리
lifecycleScopeActivity 수명UI 이벤트, bridge 핸들러

로그아웃 시 loginSessionScope.cancel()로 모든 사용자 범위 코루틴을 한 번에 정리합니다.
