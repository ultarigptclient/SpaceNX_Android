message.db 사용자별 / 쪽지

쪽지(Note)는 채팅과 분리된 1:1 또는 1:N 메시지입니다. 채팅과는 다른 sync 주기·회수(Retrieve) 기능·첨부 처리. 파일: message_{userId}.db. 정의: MessageDatabase.kt

MessageEntity

@Entity(tableName = "messages")
data class MessageEntity(
    @PrimaryKey val messageCode: String,
    val sendUserId: String = "",
    val sendUserName: String = "",
    val receivers: String = "[]",       // JSON 배열 [{userId, read:bool, readDate}]
    val title: String = "",
    val contents: String = "",
    val sendDate: Long = 0L,
    val readCount: Int = 0,
    val totalReceivers: Int = 0,
    val retrieved: Int = 0,              // 1=회수됨
    val additional: String = "{}"
)

AttachEntity

@Entity(tableName = "attaches")
data class AttachEntity(
    @PrimaryKey(autoGenerate = true) val attachId: Int = 0,
    val messageCode: String = "",
    val fileName: String = "",
    val fileSize: Long = 0L,
    val mimeType: String = "",
    val url: String = "",
    val uploadDate: Long = 0L
)

회수(Retrieve) 기능

송신자가 자기가 보낸 쪽지를 취소하는 기능. DB 상 retrieved=1로 마킹. Push 이벤트 RETRIEVE_MESSAGE_EVENT (0x0B83) 수신 시 수신자 측에서 같은 처리.
