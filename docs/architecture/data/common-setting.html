common.db + setting.db

common은 로그인 전에도 접근 가능한 전역 DB, setting은 사용자 개인 설정.

common.db 전역

파일: common.db (userId 접미사 없음). 정의: CommonDatabase.kt

Entity테이블역할
CommonEntitycommonkey-value 저장소 — 서버 설정, 버전, FCM 토큰 등
SyncMetaEntitysyncMeta도메인별 마지막 sync 시각 (예: chat_last_sync_time)

CommonEntity

@Entity(tableName = "common")
data class CommonEntity(
    @PrimaryKey val key: String,
    val value: String = ""
)

주요 키 (관례)

key용도
FRONTEND_VERSIONSPA 빌드 버전 (URL 조합)
FRONTEND_SKIN테마 스킨 ID
ultari.rest.base-urlREST 기본 URL (syncConfig 결과)
ultari.rest.endpoints.*개별 엔드포인트 경로
fcm_tokenFCM 토큰 캐시

SyncMetaEntity

@Entity(tableName = "syncMeta")
data class SyncMetaEntity(
    @PrimaryKey val key: String,
    val value: String = ""              // 대부분 Long (epoch ms) 또는 eventId
)

주요 키: chat_last_sync_time, channel_last_sync_time, message_last_sync_time, noti_last_event_id 등.

setting.db 사용자별

파일: setting_{userId}.db. 정의: SettingDatabase.kt

Entity테이블역할
SettingEntitysettings사용자 개인 설정 key-value

SettingEntity

@Entity(tableName = "settings")
data class SettingEntity(
    @PrimaryKey val key: String,
    val value: String = ""
)

사용 예

  • 사용자 UI 설정 (다크모드, 폰트크기 등)
  • 알림 토글 (채널별 뮤트는 별도 AppConfig에 보관)
  • getUserConfig / setUserConfig 브릿지 액션에서 읽고 씀
참고: AppConfig와의 차이
AppConfig (common/AppConfig.kt)는 EncryptedSharedPreferences 기반. 민감한 자격증명/토큰 저장용. 반면 setting.db는 평문 Room DB (SQLCipher로는 암호화됨)로 일반 UX 설정용.
