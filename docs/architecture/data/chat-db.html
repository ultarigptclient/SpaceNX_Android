1. ChannelEntity

@Entity(tableName = "channels")
data class ChannelEntity(
    @PrimaryKey val channelCode: String,
    val channelName: String = "",
    val masterChannelName: String = "",
    val masterUserId: String = "",          // 마스터 (방장)
    val sendUserId: String = "",            // 최근 발신자
    val channelType: String = "",           // DM / GROUP / CONFERENCE 등
    val lastChatDate: Long = 0L,
    val lastChatContents: String = "",
    val unreadCount: Int = 0,               // 채널 배지용 (단, 최근엔 offset 기반 재계산 권장)
    val additional: String = "{}"           // 확장 JSON (설정/메타)
)

채널 타입

channelType설명
DM1:1 대화
GROUP그룹 대화
CONFERENCE회의 (LiveKit)

2. ChatEntity (말풍선)

@Entity(
    tableName = "chats",
    primaryKeys = ["channelCode", "chatCode"],
    indices = [
        Index("channelCode"),
        Index(value = ["channelCode", "sendDate"])
    ]
)
data class ChatEntity(
    val channelCode: String,
    val chatCode: String,
    val sendUserId: String? = null,
    val contents: String = "",
    val sendDate: Long = 0L,
    val additional: String? = null,    // JSON — 리액션/투표/파일첨부 메타
    val chatType: Int = 0,             // 메시지 유형 (0=일반, 시스템, 파일 등)
    val systemCommand: String? = null, // 시스템 메시지 명령 (입장/퇴장 등)
    val chatFont: String = "{}",       // 폰트 스타일 JSON
    val state: Int = 0                 // 0=정상, -1=삭제됨
)
답글(Reply)은 별도 엔티티가 아님
ChatEntity에 parentChatCode나 replyTo 필드가 없습니다. 이 앱에서 "답글"은 스레드(ChatThreadEntity)를 통해 처리됩니다. 인라인 답글은 additional JSON의 메타데이터로 포함되거나, 스레드 기능으로 대체.

// ChatDao
@Query("""SELECT COUNT(*) FROM chats
    WHERE channelCode = :channelCode
      AND sendDate > :offsetDate
      AND sendUserId != :myUserId
      AND state = 0""")

5. ChatEventEntity (Push 로그)

서버 Push로 들어온 CRUD 이벤트를 기록. React가 나중에 syncChat 호출 시 이 로그를 기반으로 differential sync.
