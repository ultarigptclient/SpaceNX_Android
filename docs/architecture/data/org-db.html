DeptEntity

@Entity(tableName = "depts")
data class DeptEntity(
    @PrimaryKey val deptId: String,
    val deptName: String = "",
    val parentDeptId: String = "",      // 부모 부서 (루트는 "")
    val hasChild: Boolean = false,
    val userCount: Int = 0,
    val sortOrder: Int = 0
)

UserEntity

@Entity(tableName = "users")
data class UserEntity(
    @PrimaryKey val userId: String,
    val userName: String = "",
    val deptId: String = "",
    val deptName: String = "",
    val position: String = "",          // 직급/직책
    val email: String = "",
    val mobile: String = "",
    val photoUrl: String = "",
    val photoVersion: Long = 0L,        // 캐시 버스팅
    val userInfo: String = "{}"         // 확장 JSON
)

BuddyEntity (내 목록)

@Entity(tableName = "buddies", primaryKeys = ["buddyId", "userId"])
data class BuddyEntity(
    val buddyId: String,                // 폴더 ID (그룹)
    val userId: String,                 // 추가된 사용자
    val buddyName: String = "",
    val buddyType: String = "USER",     // USER / GROUP
    val sortOrder: Int = 0
)

주요 DAO 쿼리

Query용도
deptDao.getRootDepts()루트 부서 (parentDeptId = "")
deptDao.getByParent(parentDeptId)하위 부서 목록
userDao.getByDept(deptId)부서별 사용자
userDao.searchByKeyword(kw)이름/직급/이메일 검색
userDao.getByUserIds(ids)일괄 조회 (getUserInfo 액션에서 사용)
buddyDao.getAll()내 목록 전체

Sync 흐름

syncOrg (POST /api/org/syncorg) → DeptEntity + UserEntity 일괄 upsert.
syncBuddy (POST /api/buddy/syncbuddy) → BuddyEntity upsert. 이후 autoSubscribeUsers로 Binary SUBSCRIBE.
