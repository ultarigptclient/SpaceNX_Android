2. ApiClient (중앙 컨트롤)

data/remote/api/ApiClient.kt는 Kotlin object로 Retrofit 인스턴스를 관리하고, 인터페이스 메서드 외에 범용 postJson/파일 업로드 기능을 제공합니다.

2-1. 범용 POST

fun postJson(
    url: String,
    body: JSONObject,
    token: String? = null
): JSONObject {
    val req = Request.Builder()
        .url(url)
        .post(body.toString().toRequestBody("application/json".toMediaType()))
        .apply { if (token != null) addHeader("Authorization", "Bearer $token") }
        .build()
    val resp = okHttp.newCall(req).execute()
    return JSONObject(resp.body?.string() ?: "{}")
}

3. JWT 인증

모든 요청 헤더에 Authorization: Bearer {accessToken}. 401 수신 시 refreshToken으로 자동 재시도.

// 저장 위치
val token = appConfig.getSavedToken()    // EncryptedSharedPreferences

// 사용 (Handler에서)
val result = withContext(Dispatchers.IO) {
    ApiClient.postJson(
        appConfig.getEndpointByPath("/comm/sendchat"),
        body,
        appConfig.getSavedToken()
    )
}

4. Endpoint Resolution

실제 사용되는 URL은 3단계 fallback으로 결정됩니다:

  1. AppConfig.configCache["ultari.rest.endpoints.{name}"] (syncConfig 결과)
  2. AppConfig.configCache["ultari.rest.base-url"] + 기본 경로
  3. strings.xml의 hardcoded 기본값
// 도움 함수
appConfig.getEndpointByPath("/comm/sendchat")
// → https://neo.ultari.co.kr:18019/api/comm/sendchat

6. 범용 NeoSend / apiPost

SPA가 전용 핸들러가 없는 임의의 경로를 호출할 때 사용. NeoSendHandler가 /api/{path}로 POST 후 응답을 그대로 Web에 전달.

// JS 측
await Bridge.apiPost('/comm/getprojects', {userId: 'u1'});
await Bridge.neoSend('SendChat', {channelCode, contents});
await Bridge.httpRequest('GET', 'https://...', headers);

7. 개발 주의

  • SSL 검증 비활성화 (Trust Manager stub) — 배포 전 정책 결정 필요
  • Retrofit이 아닌 postJson으로 대부분 호출 — 타입 안전성이 약함
  • 에러 응답은 {errorCode:N, errorMessage:"..."} 포맷. N=0 이 성공
